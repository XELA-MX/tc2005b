Comprar Productos

Con JavaScript y HTML5 [y quizás CSS], desarrolla una página para vender 3 productos de tu interés, con los precios y promociones a tu gusto. La página debe permitir al usuario escoger la cantidad de unidades de cada producto, y debe mostrar el precio total, el IVA que se está cargando, y toda la información que consideres pertinente para que la experiencia del usuario sea la mejor. ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Porque es una buena práctica validar los inputs antes de enviarlos al servidor, ya que esto evita que se envíen datos incorrectos o mal formateados, lo que puede causar errores en el servidor y en la base de datos. Además, al validar los inputs en el lado del cliente, se mejora la experiencia del usuario al proporcionar retroalimentación inmediata sobre los errores de entrada.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Puedes saltarte la seguridad de las validaciones hechas con JavaScript deshabilitando JavaScript en el navegador o utilizando herramientas como Postman para enviar solicitudes HTTP directamente al servidor sin pasar por la interfaz de usuario. También puedes modificar el código JavaScript en el navegador utilizando las herramientas de desarrollo del navegador para omitir las validaciones.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Porque aunque las validaciones de JavaScript pueden ser eludidas, siguen siendo una buena práctica para mejorar la experiencia del usuario y reducir la carga en el servidor. Las validaciones del lado del cliente son una primera línea de defensa que puede ayudar a prevenir errores comunes y mejorar la usabilidad de la aplicación. Sin embargo, siempre se deben realizar validaciones adicionales en el lado del servidor para garantizar la seguridad y la integridad de los datos.
